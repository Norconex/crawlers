# DomSplitter

## Summary

Splits HTML, XHTML, or XML document on elements matching a given selector.

## Notes

This option builds a full DOM tree from the document content in memory. For large files, be cautious—this can use a lot of memory. For better performance with big files, consider a streaming approach like (e.g., [XMLStreamSplitter](http://localhost:3000/en/crawler-doc/web#crawler.importer.handlers.handler.XmlStreamSplitter)).

The [jsoup](http://jsoup.org/) parser library is used to load a document content into a DOM tree. Elements are referenced using a [CSS or JQuery-like syntax](http://jsoup.org/cookbook/extracting-data/selector-syntax).

### Content-types

By default, this filter applies only to documents with content types specified by the [CommonRestrictions.domContentTypes(String)](https://opensource.norconex.com/importer/v3/apidocs/com/norconex/importer/handler/CommonRestrictions.html#domContentTypes(java.lang.String)) method. You can also define your own content types if they represent files containing HTML or XML-like markup tags.

#### Common content-types:
- application/atom+xml
- application/mathml+xml
- application/rss+xml
- application/vnd.wap.xhtml+xml
- application/x-asp
- application/xhtml+xml
- application/xml
- application/xslt+xml
- image/svg+xml
- text/html
- text/xml

### Parser Handler Compatibility

This functionality can be applied as either a `pre-parse` or `post-parse` handler.

### Options:

- `ignoreCase` with pattern "Paul" will match both "Paul" and "paul". 
- `ignoreDiacritic` with the pattern "cafe" will match both "café" and "cafe".  

- `matchEmpty` specifies that `null` or `empty` strings are treated as positive matches.

- `method` defines the matching strategy (`BASIC`, `WILDCARD`, `CSV` or `REGEX`) with correlated `pattern`. Match/replace methods are case-sensitive by default. 

- `negateMatches` excludes target that match the pattern and includes all others.

- `partial` with the pattern "Paul" will match "Paul" in the sentence "I am Paul."

- `pattern` specifies the text or pattern used for matching fields, with its behavior determined by the selected `method`. If no pattern is provided (i.e. the pattern is null), the matcher is treated as matching all input text by default, but it will not perform any replacements 
      
- `replaceAll` replacing all instances of the `pattern` in the field, not just the first match; for example, replacing "foo" bu "bar" in "foo and foo" results in "bar and bar".

- `trim` leading and trailing spaces are removed from the field before matching; for example, "Paul" will match successfully even if the field contains " Paul ".

#### Empty and null values
Null or empty strings are not matched by default. To have those considered as positive matches, set matchEmpty to true. To have blank values (containing white spaces only) considered as positive matches, also set trim to true. When matching empties, doing replacement on a null value behaves as if the value is an empty string.

### discardOriginal

Discard the original document after the split is performed.

### fieldMatcher

Matches document fields based on specified patterns and methods

### parser

Defines using XML or HTML parser.

### selector

XPath expression for splitting nodes.

### sourceCharset

Sets the character encoding of the source document to ensure proper reading and processing.

## Examples

The following example splits contant found in an HTML document, each one being stored within a div with a class named "contact".

<ConfigExample
  json={{
    "handler": {
      "class": "DomSplitter",
      "selector": "div.contact"
    }
  }}
/>

### content Type

The following example splits content from a field named `embedded_file`, but only when the content type is `text/html`. It targets HTML elements matching the `div.contact` selector within that field

<ConfigExample
  json={{
    "handler": {
      "class": "DomSplitter",
      "selector": "div.contact",
      "parser": "HTML",
      "contentTypeMatcher": {
        "method": "BASIC",
        "pattern": "text/html"
      },
      "fieldMatcher": {
        "method": "BASIC",
        "pattern": "embedded_file"
      }
    }
  }}
/>
