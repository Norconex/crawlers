# DefaultParser

## Summary

Generic document parser for raw content passthrough or text decoding

## Notes

It uses Apacke Tika for most of its supported content types. For unknown content types, it falls back to Tika generic media detector/parser.

### Embedded

By default, embedded documents (e.g., attachments in emails, files in ZIP archives) are merged into their container (parent) document. You can configure the parser to split embedded documents, allowing each to be processed individually through the full import cycle. You can also control whether embedded documents should be skipped during processing.

#### Options

**splitContentTypes**: Defines which container documents should have their embedded documents split into individual documents.

>Example: Splits embedded documents found in ZIP archives or email messages.*
- ```xml
  <pattern>application/zip|message/rfc822</pattern>
  ``` 


**skipEmbeddedContentTypes**: Specifies embedded document types that should not be extracted from their containers. Skipping embedded documents depends on the child content type itself

>*Example: Skips extracting image files and PDFs from any container.*
- ```xml
  <pattern>image/.*|application/pdf</pattern>
  ```


**skipEmbeddedOfContentTypes**: Specifies container document types from which embedded documents should not be extracted. Skipping embeded documents base on the container types.

  >*Example:Prevents extracting embedded documents from Microsoft Word files and still process the container, Word document, itself*
  - ```xml
    <pattern>application/msword</pattern>
    ```

### Optical character recognition (OCR):

You can configure this parser to use the [Tesseract](https://code.google.com/p/tesseract-ocr/) open-source OCR application to extract text out of images or documents containing embedded images (e.g. PDF). Supported image formats are TIFF, PNG, JPEG, GIF, and BMP.

To enable this feature, you must first download and install a copy of Tesseract appropriate for your platform (supported are Linux, Windows, Mac and other platforms). It will only be activated once you configure the path to its install location. Default language detection is for English. To support additional or different languages, you can provide a list of three-letter ISO language codes supported by Tesseract. These languages must be part of your Tesseract installation. You can [download additional languages](https://code.google.com/p/tesseract-ocr/downloads/list) form the Tesseract web site.

When enabled, OCR is attempted on all supported image formats. To limit OCR to a subset of document content types, configure the corresponding content-types (e.g. application/pdf, image/tiff, image/png, etc.).

## Examples

Save Error Parse to a directory

<ConfigBlock
  json={{
    "handler": {
      "class": "DefaultParser",
      "errorsSaveDir": "C:\\reports\\errors"
    }
  }}
/>

### ocr

The following example uses Tesseract to convert English images in PDF into text.

***OCR***

<ConfigBlock
  json={{
    "handler": {
      "class": "DefaultParser",
      "ocr": {
        "tessdataPath": "/app/ocr/tesseract.exe",
        "contentTypeMatcher": {
          "method": "BASIC",
          "pattern": "application/pdf",
          "ignoreCase": true
        },
        "language": "eng"
      }
    }
  }}
/>

### embedded

The following example separates embedded content into individual documents for ZIP files, skips extracting image-type content (e.g., PNG, JPEG) from all container types—including ZIPs, and entirely skips embedded content when the parent document is a Microsoft Word file.

***Embedded***



<ConfigBlock
  json={{
    "handler": {
      "class": "DefaultParser",
      "embedded": {
        "splitContentTypes": [
          {
            "method": "BASIC",
            "pattern": "application/zip",
            "ignoreCase": true
          }
        ],
        "skipEmbeddedContentTypes": [
          {
            "method": "REGEX",
            "pattern": "image/.*",
            "ignoreCase": true
          }
        ],
        "skipEmbeddedofContentTypes": [
          {
            "method": "BASIC",
            "pattern": "application/msword",
            "ignoreCase": true
          }
        ]
      }
    }
  }}
  xml={`
<handler>
  <class>DefaultParser</class>
  <embedded>
    <splitContentTypes>
      <splitContentType>
        <method>BASIC</method>
        <pattern>application/zip</pattern>
        <ignoreCase>true</ignoreCase>
      </splitContentType>
    </splitContentTypes>
  </embedded>
</handler>`}
/>
