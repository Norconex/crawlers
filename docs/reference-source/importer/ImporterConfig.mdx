# ImporterConfig

## Summary

Configures the document importing pipeline with handlers for parsing, transforming, and filtering documents.

## Notes

Configures the document importing pipeline. Handlers are executed in order and can include transformers, parsers, splitters, and flow control wrappers. Handlers applied before a parser run as pre-parse handlers; those after run as post-parse handlers. Including a parser transitions the pipeline from pre-parse to post-parse mode.

## Properties

### handlers

Specify handlers for imported documents in their original format or applied before or after parsing. Combine with condition wrappers to create a processing "flowControl"

### Transformer
- Transformers are used to manipulate or annotate crawled data before it is stored or processed further. They can add metadata, modify existing fields, or remove unnecessary information to prepare the data for downstream applications.

- Transformers can be applied at various stages, such as after parsing or before storage, making them versatile tools in data enrichment and normalization.

### Parser
- The parser processes the raw content of documents to extract meaningful data. It converts the raw bytes of a document into structured or semi-structured data by recognizing its format and interpreting its content.

- Parsers can be customized to focus on specific document elements, such as extracting only the main content of a webpage or handling complex formats with external libraries.

### Splitter
- The splitter breaks down large or complex documents into smaller, manageable segments for more targeted processing or storage. This is particularly useful for documents that contain multiple sections, such as a webpage with several articles or a PDF with multiple chapters.

- Some splitters are designed for use in the preprocessing stage, while others can be applied in both preprocessing and postprocessing; refer to the Splitter Documentation for more details.

### Flow Control

- Flow control mechanisms manage the sequence and conditions under which various operations are executed during the crawling process.

- Flow control can be applied both in preprocessing and postprocessing stages.

## Examples

The following example demonstrates a flexible handler configuration that processes data based on its content type and metadata. 

First, the conditional checks the `Content-Type` field to determine if the content is HTML. It rejected using the `Reject` handler, if the content is HTML. Otherwise, the `DefaultParser` is applied to parse the non-HTML data. 

Following parsing, another conditional handler evaluates the `date_created` field using a `DateCondition`. If the date is within the range, the content is committed to an Elasticsearch index named `new` using the `ElasticsearchCommitter`. If the date does not meet the condition, the content is committed to an Elasticsearch index named `archive`.
